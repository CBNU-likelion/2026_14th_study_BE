## 로깅 시스템 구현 상세

본 프로젝트는 시스템 모니터링과 디버깅 편의성을 위해 다음과 같이 로깅 시스템을 구축하였습니다.

### 1. HTTP 요청/응답 로깅 (Filter)
- **적용 방식**: `OncePerRequestFilter`를 상속받은 `LoggingFilter` 사용
- **선택 이유**: HTTP 요청의 시작부터 끝까지 전체 생명주기를 관장하기에 가장 적합하며, Controller에 도달하기 전/후의 정보를 일관되게 획득할 수 있습니다.
- **로깅 내용**: HTTP 메서드, 요청 URI, 응답 상태 코드

### 2. 실행 시간 측정 (AOP)
- **적용 방식**: 커스텀 어노테이션 `@LogExecutionTime`과 Spring AOP 사용
- **선택 이유**: 특정 로직의 실행 시간을 측정할 때 비즈니스 로직을 오염시키지 않고 선언적으로 적용할 수 있습니다.
- **대상**: `@LogExecutionTime`이 부착된 모든 메서드

### 3. 예외 로깅 (AOP)
- **적용 방식**: `@AfterThrowing` 어드바이스 사용
- **선택 이유**: `GlobalExceptionAdvice`는 클라이언트 응답 생성에 집중하고, AOP는 시스템 내부 로깅에 집중하여 역할을 분리하였습니다. 이를 통해 비즈니스 코드 내 별도의 `try-catch` 없이도 예외 상황을 기록할 수 있습니다.
- **대상**: `com.likelion.backend.api.article` 패키지 내 모든 메서드
